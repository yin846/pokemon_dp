WM_Enable

C Specification

#include <nitro/wm.h>

WMErrCode WM_Enable( WMCallbackFunc callback );

Arguments

callback Designates the callback function to call when the asynchronous process completes. This callback is called from within the interrupt handler.

Return Values

Returns the results of the WMErrCode enumerated type process.

Description

Enables wireless hardware. If this function's return value is WM_ERRCODE_OPERATING, a callback will be called after the wireless hardware enable process completes. This function can only be called when the WM is in the READY state. After completion, the WM will move to the STOP state. To start communications after this function is called, you must use WM_PowerOn to start the power to the wireless hardware.

See Also

WM_Init, WM_PowerOn, WM_PowerOff, WM_Disable, WM_Finish, WM_Initialize, WM_End
