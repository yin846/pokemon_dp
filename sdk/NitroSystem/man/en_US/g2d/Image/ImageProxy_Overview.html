Image Proxy Overview

Conceptual Diagram

Description

With the G2D Library, you can use the same API to dynamically switch between display with the main screen OBJ, the sub-screen OBJ, and the software sprite. However, to actually carry out rendering for each of these displays, character data needs to be stored for each in a separate VRAM slot. A feature named ImageProxy exists in the G2D Library so you do not have to be concerned about switching the character data storage destination when the display destination changes. If the base address of the character data is requested from the user, ImageProxy determines how the character display is supposed to be displayed and returns the character base address stored in the appropriate VRAM slot. This organization is used for operations such as setting up VRAM information in upper utility level renderer modules.
